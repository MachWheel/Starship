# Brazilian-Portuguese voice manifest for Star Fox 64 (Starship port).
# Loaded at runtime by audio_general.c::load_voice_manifest. Edit and restart
# the game — no rebuild needed.
#
# Format (pipe-separated):
#     <msgIds> | <letSeqRunForMsgIds> | <wavPath>
#
#   msgIds                  Comma-separated voice msgIds that map to this WAV.
#                           Multiple msgIds let one line span multiple
#                           Audio_PlayVoice calls (e.g., Pepper's intro fires
#                           on msgId=3, displayable text is msgId=1200).
#
#   letSeqRunForMsgIds      Comma-separated subset of msgIds that should still
#                           dispatch to the seq player (lets the radio click
#                           SFX baked into the original sample play). Use "-"
#                           if the seq should never run for this line.
#
#   wavPath                 Forward-slash relative path to a 32 kHz mono
#                           16-bit PCM .wav file, resolved against the
#                           process working directory.
#
# Lines starting with # and blank lines are ignored. Whitespace is permitted
# around tokens.

# ==== Intro (Pepper briefs Star Fox at game open, before prologue scroll) ====
#
# All 6 intro msgIds use letSeqRun=- so the original seq voice is never
# dispatched — zero possibility of EN voice bleed. The radio key-up /
# key-down clicks are baked directly into 3 specific WAV files (mixed in
# externally, see ptbr_audio/radio/start.wav and end.wav for the source
# samples): 60/10 have start prefix, 60/50 have end suffix. 20/30/40 are
# mid-Pepper-transmission with no click overlay.
#
# Optional 4th field "endinwav": tells Audio_PlayVoice to skip the seq
# voice 0 dispatch that fox_radio.c case 6 fires after a transmission ends
# (which would normally play the game's baked-in key-down click). Set on
# the lines whose dub WAV already includes the mixed-in end click — rob/60
# and pepper/50 — to avoid stacking two end clicks on top of each other.
